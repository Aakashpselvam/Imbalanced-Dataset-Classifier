# Imbalanced-Dataset-Classifier
# Overview
This project focuses on building a machine learning model to handle imbalanced datasets in domains such as fraud detection and medical diagnosis. The goal is to accurately identify minority class instances, which are often overlooked by traditional models.

The project also includes a complete evaluation dashboard to visualize model performance using advanced metrics.


# Aim
To develop a robust classifier for imbalanced datasets and evaluate its performance using metrics like Confusion Matrix, ROC-AUC, Precision-Recall Curve, F1 Score, and IoU, along with a visualization dashboard.


# Dataset
- Imbalanced dataset with majority and minority classes  
- Contains features such as transaction details or medical records  
- Reflects real-world class imbalance scenarios  



# Technologies Used
- Python  
- NumPy, Pandas  
- Scikit-learn  
- Matplotlib, Seaborn  
- (Optional) Streamlit for dashboard  



# Workflow
1. Data Collection  
2. Data Preprocessing (Cleaning, Encoding, Normalization)  
3. Handling Imbalance (Oversampling / Undersampling / SMOTE)  
4. Model Training (Logistic Regression / Random Forest / Neural Network)  
5. Model Evaluation  
6. Dashboard Visualization  


#  Evaluation Metrics
- Confusion Matrix  
- ROC Curve & AUC Score  
- Precision-Recall Curve  
- F1 Score  
- IoU (Jaccard Score)  

# Results
The model performance is evaluated using multiple metrics to ensure balanced classification, especially for the minority class.  
Visualizations such as confusion matrix, ROC curve, and precision-recall curve help in understanding model behavior.
